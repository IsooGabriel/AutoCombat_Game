# AutoCombat Game

## プロジェクトの概要 (Project Overview)
本プロジェクトは、ノードベースのグラフエディタを用いてキャラクターのAI（行動アルゴリズム）を構築し戦わせる、2Dパズルアクション・オートバトルゲームです。
単なるゲームにとどまらず、基盤となるグラフシステムは「汎用的なアルゴリズム構築プラットフォーム」として設計されており、MOD（カスタムノード）の導入によってスクレイパーなどの非ゲーム用途にも転用可能な拡張性を備えています。

## 主な機能と実装詳細 (Features and Implementation Details)

### 1. グラフシステム (Graph System)
プレイヤーがキャラクターの行動を視覚的にプログラミングするためのコアシステムです。

- **独立した実行サイクル**: `GraphExecutor.cs` および `GraphRunner.cs` を通して、Unityの標準ライフサイクル(`Update`)とは独立した **30Hz (30Tick/sec)** の固定周期で動作します。これにより、フレームレートに依存しない安定した挙動と、バックグラウンドでの高速シミュレーションを可能にしています。
- **ノードの拡張と多様性**: `Node.cs` を基底クラスとして、`MoveNode`, `AttackNode`, `IfNode`, `ANDNode`, `SumNode` などの豊富な制御・アクションノードが実装されています。
- **JSONによるデータ管理**: `GraphData.cs` と `GraphDataJsonUtility.cs` を用いて、作成したグラフはJSON形式でシリアライズ・保存されます。この設計により、後方互換性を保ちつつ、Webサイト上での共有や非同期オンライン対戦などの機能拡張を容易にしています。
- **汎用拡張性 (MOD対応)**: `GenericNode.cs` と `CustomNodeRegistry` を用いることで、JSON定義から動的にカスタムノードを生成・登録でき、ソースコードを直接変更することなく機能を拡張（MOD化）できるようになっています。

### 2. キャラクター＆戦闘システム (Character & Combat System)
グラフシステムからの命令を受け取り、ゲーム内で物理的・ステータス的な処理を行います。

- **ステータス管理**: `Character.cs` によって管理され、キャラクターの基本ステータス (`baseStatus`) と、装備やバフによる追加ステータス (`aditionalStatus`) を合算して最終的な能力値を計算する柔軟な仕組みとなっています。
- **武器システム**: `Weapon.cs` を基底とし、`WeaponSelector.cs` で管理されます。各武器は固有のダメージ倍率 (`DamageMultiply`) や挙動を持ち、ゲームバランスの調整や新しい武器の追加が容易な設計になっています。

### 3. システムアーキテクチャ (System Architecture)
持続可能な開発と安全なコードベースを維持するための設計思想が取り入れられています。

- **依存性の注入 (DI)**: シングルトンパターンによる密結合を避け、`VContainer` を用いたDI (`GameLifetimeScope.cs`) を段階的に導入しています。これにより、各マネージャークラスのテスト容易性と安全性が向上しています。
- **拡張優先の原則**: 既存のロジックを直接書き換えるのではなく、インターフェースや継承を用いて「拡張」することで機能を追加する方針をとっています。
